The one mental model to hold onto

Almost all of web development is one simple loop. A client (the browser) asks for something. A server answers. Everything else — HTML, CSS, React, Node, databases, Redis — is just detail hanging off those two words. Keep that picture in your head and nothing below will feel random.

frontend
Everything the user sees and clicks, running in the browser. Built with HTML, CSS and JavaScript (and React).
backend
The engine behind the scenes on a server: logic, rules, and talking to the database. Built with Node/Express here.
full-stack
You, by the end of this path — comfortable on both sides and able to wire them together.
request → response
The heartbeat of the web. Browser sends a request, server sends back a response. Repeat forever.
